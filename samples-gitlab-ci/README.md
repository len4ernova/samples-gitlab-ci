Репозиторий включает коллекцию pipelenes GitLab.
Хороший pipelines (конвеер) обычно преследует одну из следующих целей:
- ориентация на разработчика, т.е. разработчик будет поддерживать, наполнять - активно "мониторить" - develops_type;
- ориентация на универсальность, подходит для множества типовых проектов, с предварительно безупречно отработанному выверенному CI - universal_type;
- комбинирование - combi_type.
develops_type, universal_type, combi_type - примеры приведены в соответствущих директориях.

+/-:
- develops_type - выполняемы шаги прописываем явно. 
    `+` - нет магии, разработчик видит состав команд;
    `-` - длина файла растет с ростом включаемых stage; переиспользование - через copy/past.
- universal_type - удобно реализовать через компоненты, т.е. переиспользуемые stage выносим в отдельные файлы. Pipeline становится похож на лего.
    `+` - один раз написал кусочки и переиспользуй;
    `-` - разработчики тратят время на вникание (в лучшем случае) или не тратят => возникает магия;
    `-` - забываются нюансы => компоненты должны быть максимально простые.
- combi_type - комбинируем. Например, "поиск забытых секретов" - универсален --> в шаблон, сборка - может иметь нюансы --> прописать явно.

P.S.: Auto DevOps - хорош, если продуман.
